TypeScript is playing the "strict" card because of a simple array mismatch in your middleware.

### The Bug

In `src/middlewares/role.middleware.ts`, your function signature is:
`export const authorize = (roles: string[]) => { ... }`

It expects an **array** of strings (e.g., `["ADMIN"]`), but in your routes, you are passing a single **string**: `authorize("ADMIN")`.

---

### The Fix: `./src/routes/order.routes.ts`

You just need to wrap the role in brackets. Also, notice that you have a route collision: both `getOrderHistory` and `getAllOrders` are using `GET /`. We should probably give the Admin route a specific path to avoid confusion.

```ts
// Admin Routes
// We changed "/" to "/admin" to avoid clashing with the customer's history route
router.get("/admin", authenticate, authorize(["ADMIN"]), getAllOrders);
router.patch("/:id/status", authenticate, authorize(["ADMIN"]), updateStatus);
```

---

### Why the Route Change?

In your current file:

1. `router.get("/", authenticate, getOrderHistory);`
2. `router.get("/", authenticate, authorize(["ADMIN"]), getAllOrders);`

Express matches routes top-down. If an Admin hits `/`, they will trigger `getOrderHistory` (their own orders) and never reach `getAllOrders`. By using `/admin` or a similar sub-path, you keep the logic clean.

---
