Two things are happening here: TypeScript is complaining about a potential `undefined` user, and your test is failing because your `adminToken` belongs to a standard `ADMIN`, but the new logic requires a `SUPER_ADMIN` to change roles.

Here is how we fix both.

### 🛠️ 1. Fix the Controller (TypeScript Error)

The `req.user` error happens because TypeScript doesn't know that your `authenticate` middleware has already run. We can fix this with a "Guard" or a type assertion.

Update `changeUserRole` in `src/controllers/user.controller.ts`:

```typescript
export const changeUserRole = async (
  req: Request,
  res: Response,
  next: NextFunction,
) => {
  try {
    // 🛡️ Fix TS18048: Check if user exists first
    if (!req.user) {
      return res.status(401).json({ error: { message: "Unauthorized" } });
    }

    // Now TS knows req.user is defined
    if (req.user.role !== "SUPER_ADMIN") {
      return res.status(403).json({
        error: { message: "Only Super Admins can manage user roles." },
      });
    }

    const { id } = req.params;
    const { role } = req.body;

    if (!["SUPER_ADMIN", "ADMIN", "CUSTOMER"].includes(role)) {
      return res
        .status(400)
        .json({ error: { message: "Invalid role provided" } });
    }

    const updatedUser = await UserService.updateUserRole(id, role);
    res.json({ message: `User role updated to ${role}`, user: updatedUser });
  } catch (error) {
    next(error);
  }
};
```
