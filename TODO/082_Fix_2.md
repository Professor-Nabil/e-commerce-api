The issue is clear! Look closely at how your `CreateProductSchema` is defined compared to how your `validate` middleware parses the data.

### The Bug

In `product.schema.ts`, you have:

```ts
export const CreateProductSchema = z.object({
  body: z.object({
    // <--- The schema expects an object with a 'body' key
    name: z.string(),
    // ...
  }),
});
```

But in your `PATCH` test, you are sending just `{ price: 150 }`. When Zod runs `.partial()` on the **outer** object, it makes the `body` key optional, but if the `body` key _is_ present (which it is, because your middleware passes `req.body`), it still expects the **full** content inside that body.

### The Fix

You need to reach inside and make the **inner** body object partial. Update your `./src/schemas/product.schema.ts` like this:

```ts
// Original Create Schema
export const CreateProductSchema = z.object({
  body: z.object({
    name: z.string().min(3),
    description: z.string().min(10),
    price: z.coerce.number().positive(),
    stock: z.coerce.number().int().nonnegative(),
  }),
});

// CORRECT Update Schema:
// We target the 'body' property specifically to make its internal fields optional
export const UpdateProductSchema = z.object({
  body: CreateProductSchema.shape.body.partial(),
});
```

---

### Why this works

1. **`CreateProductSchema.shape.body`**: This extracts the Zod object that defines the fields (`name`, `price`, etc.).
2. **`.partial()`**: This makes every field inside that specific object optional.
3. **Wrapping in `z.object({ body: ... })`**: This puts it back into the structure your `validate` middleware expects (where it validates `req.body` against the `body` key of the schema).
