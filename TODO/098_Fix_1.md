You’ve got a solid foundation here, but since we just implemented the `AppError` architecture, your `order.service.ts` and `order.controller.ts` are a bit "old school." They are still using generic errors and manual status checks in the controller.

Let’s refactor this to use the **Custom Class** approach we agreed on. This will clean up your controllers and make the transaction logic more robust.

### 1. Refactor: `./src/services/order.service.ts`

We need to swap generic errors for `AppError`. This ensures that if a transaction fails due to stock, it automatically carries a `400` status.

```ts
import { prisma } from "../config/prisma.js";
import { AppError } from "../utils/appError.js";

export const checkout = async (userId: string) => {
  return await prisma.$transaction(async (tx) => {
    // 1. Get the cart and items
    const cart = await tx.cart.findUnique({
      where: { userId },
      include: { items: { include: { product: true } } },
    });

    if (!cart || cart.items.length === 0) {
      throw new AppError("Cart is empty", 400); // Clean 400
    }

    // 2. Validate stock and calculate total
    let total = 0;
    for (const item of cart.items) {
      if (item.product.stock < item.quantity) {
        throw new AppError(`Not enough stock for ${item.product.name}`, 400);
      }
      total += Number(item.product.price) * item.quantity;
    }

    // 3. Create the Order
    const order = await tx.order.create({
      data: {
        userId,
        totalAmount: total,
        status: "COMPLETED",
        items: {
          create: cart.items.map((item) => ({
            productId: item.productId,
            quantity: item.quantity,
            price: item.product.price,
          })),
        },
      },
    });

    // 4. Update Stock (Atomically)
    for (const item of cart.items) {
      await tx.product.update({
        where: { id: item.productId },
        data: { stock: { decrement: item.quantity } },
      });
    }

    // 5. Clear the Cart
    await tx.cartItem.deleteMany({ where: { cartId: cart.id } });

    return order;
  });
};
```

### 2. Refactor: `./src/controllers/order.controller.ts`

Look how much cleaner this becomes. We no longer need to "sniff" the error message to decide the status code; the `errorHandler` middleware handles it via the `AppError` class.

```ts
import { Request, Response, NextFunction } from "express";
import * as OrderService from "../services/order.service.js";

export const checkout = async (
  req: Request,
  res: Response,
  next: NextFunction,
) => {
  try {
    const userId = req.user!.id;
    const order = await OrderService.checkout(userId);
    res.status(201).json(order);
  } catch (error) {
    next(error); // Just pass it along!
  }
};

export const getOrderHistory = async (
  req: Request,
  res: Response,
  next: NextFunction,
) => {
  try {
    const userId = req.user!.id;
    const orders = await OrderService.getOrderHistory(userId);
    res.status(200).json(orders);
  } catch (error) {
    next(error);
  }
};
```

### 3. Bug Fix: Decimal Precision in `totalAmount`

In your service:
`total += Number(item.product.price) * item.quantity;`

If `price` is a Prisma `Decimal`, using `Number()` can sometimes lead to floating-point precision issues (e.g., $19.99 + $0.01 = 20.00000000004$). Since you are dealing with money, it's safer to keep the math robust. If your database field for `totalAmount` is a Decimal/Float, Prisma handles the output fine, but keep an eye on this if you start doing complex discounts.
