The mystery is solved! The smoking gun is right here:
`DEBUG: Product ID from response: undefined`

Your test is failing because it's trying to add `undefined` to the cart. Since your Zod schema expects a string (specifically a UUID), it throws a `400 "Required"` error because the `productId` is missing from the request body.

### 🔍 Why is it `undefined`?

In your test setup:

```typescript
const prod = await request(app)
  .post("/api/products")
  .set("Authorization", `Bearer ${adminLogin.body.token}`)
  .send({ ... });

const productId = prod.body.id; // This is coming back undefined!
```

There are two likely reasons for this:

1. **The Response Object Structure**: Sometimes the product isn't at the root of `res.body`. If your controller sends `{ data: product }`, then you need `prod.body.data.id`.
2. **Middleware/RBAC Blocking**: If the admin login failed or the `authorize(["ADMIN"])` middleware blocked the request, `prod.statusCode` might be `403` or `401`, meaning no product was created and the body contains an error message instead of a product object.

### 🛠️ The Fix: Strict Testing

Update your `tests/checkout.test.ts` to catch these issues immediately before they cascade into the checkout logic:

```typescript
// 1. Setup Admin & Product
const adminLogin = await request(app)
  .post("/api/auth/login")
  .send({ email: "admin_cart@test.com", password: "password123" });

// SAFETY: Check if admin login actually worked
expect(adminLogin.statusCode).toBe(200);

const prod = await request(app)
  .post("/api/products")
  .set("Authorization", `Bearer ${adminLogin.body.token}`)
  .send({
    name: "GPU",
    description: "RTX 5090 Super",
    price: 2000,
    stock: 10,
  });

// SAFETY: Check if product was created. If this fails, look at your product controller!
expect(prod.statusCode).toBe(201);

const productId = prod.body.id;
console.log("DEBUG: Confirmed Product ID:", productId);

// Now the rest will work
```
