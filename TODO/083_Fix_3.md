### The Fix: Match the Schema to your Test

You have two choices: change the test to match Zod's default, or change Zod to speak the way your test expects. Usually, providing specific error messages in Zod is better for your frontend partner.

Update your **`./src/schemas/product.schema.ts`** to include the custom message:

```ts
import { z } from "zod";

export const CreateProductSchema = z.object({
  body: z.object({
    name: z.string().min(3),
    description: z
      .string()
      .min(10, "Description must be at least 10 characters"), // <-- Add this
    price: z.coerce.number().positive(),
    stock: z.coerce.number().int().nonnegative(),
  }),
});

export const UpdateProductSchema = z.object({
  body: CreateProductSchema.shape.body.partial(),
});
```

### Why this is actually good

By explicitly adding `"Description must be at least 10 characters"`, you ensure that your API returns friendly, human-readable errors rather than technical strings like `"String must contain at least 10 character(s)"`.
