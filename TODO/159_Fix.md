The issue is that Vitest's `toBeGreaterThan` expects a number or bigint, but Prisma returns a `Date` object for the `resetTokenExpires` field. We just need to convert the dates to their millisecond timestamps using `.getTime()`.

```ts
  // Convert both to timestamps (numbers) to satisfy TypeScript
  const expiryTime = user?.resetTokenExpires?.getTime();
  const currentTime = new Date().getTime();

  expect(expiryTime).toBeGreaterThan(currentTime);
});
```

---

### 📝 Why this happened?

In JavaScript/TypeScript, when you compare two Date objects using `>` or `<` in a standard `if` statement, JS implicitly calls `.valueOf()`. However, Vitest's `toBeGreaterThan` matcher is strictly typed to only accept `number | bigint`.

By calling `.getTime()`, we:

1. **Satisfy TS**: It returns a `number`.
2. **Explicit Comparison**: We are comparing the Unix timestamp (ms), which is more robust.
