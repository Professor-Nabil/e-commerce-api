The error `"Expected array, received string"` is happening because of how Swagger (and `curl -F`) sends empty or single fields versus how your **Zod schema** expects them.

When you leave `categoryNames` empty in Swagger, it sends `categoryNames=""` (a string). Your Zod schema expects an array: `categoryNames: z.array(z.string())`.

### 🛠️ The Fix: Update the Zod Schema

We need to make the schema flexible enough to handle a single string and turn it into an array automatically. This is a common issue with `multipart/form-data`.

Update your `./src/schemas/product.schema.ts`:

```typescript
import { z } from "zod";

export const CreateProductSchema = z.object({
  body: z.object({
    name: z.string().min(3),
    description: z.string().min(10),
    price: z.coerce.number().positive(),
    stock: z.coerce.number().int().nonnegative(),
    // 💡 Fix: Pre-process the input to handle strings or empty values
    categoryNames: z
      .preprocess((val) => {
        if (!val || val === "") return []; // Handle empty string from Swagger
        if (typeof val === "string") return [val]; // Convert single string to array
        return val;
      }, z.array(z.string()))
      .optional(),
  }),
});
```

### 🔍 Why Swagger failed but your manual `curl` worked?

1. **Empty Fields:** In your successful manual `curl`, you provided two `-F "categoryNames=..."` flags. Multer saw multiple keys and created an array.
2. **Swagger's Behavior:** When you click "Execute" in Swagger with only one category (or an empty one), it sends a single key. Without the `preprocess` logic, Zod sees `categoryNames: "Electronics"` and says: _"Hey, I wanted an array `["Electronics"]`, not a string!"_
