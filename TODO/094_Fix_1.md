I see the logic is already in `addToCart`, but your `updateQuantity` is currently flying blind—it updates the database without checking if you actually have that much stock. Also, your current `addToCart` has a potential bug: using `increment: quantity` on a product that is already in the cart might push the total quantity above the available stock.

Let's fix the logic in the service first, then we'll handle the global error mapping so these "Insufficient stock" errors return a **400** instead of a **500**.

### 1. The Service Fix: `./src/services/cart.service.ts`

We need to validate stock in `updateQuantity` and fix the `increment` logic in `addToCart`.

```ts
// Update to addToCart logic inside the file
export const addToCart = async (
  userId: string,
  productId: string,
  quantity: number,
) => {
  if (!userId) throw new Error("User ID is required");

  let cart = await prisma.cart.findUnique({ where: { userId } });
  if (!cart) cart = await prisma.cart.create({ data: { userId } });

  const product = await prisma.product.findUnique({ where: { id: productId } });
  if (!product || product.isDeleted) throw new Error("Product not found");

  // Check existing item to see total requested quantity
  const existingItem = await prisma.cartItem.findUnique({
    where: { cartId_productId: { cartId: cart.id, productId } },
  });

  const totalRequested = (existingItem?.quantity || 0) + quantity;
  if (product.stock < totalRequested) {
    throw new Error(
      `Insufficient stock. Only ${product.stock} items available.`,
    );
  }

  return await prisma.cartItem.upsert({
    where: { cartId_productId: { cartId: cart.id, productId } },
    update: { quantity: { increment: quantity } },
    create: { cartId: cart.id, productId, quantity },
  });
};

// Update to updateQuantity logic
export const updateQuantity = async (
  userId: string,
  productId: string,
  quantity: number,
) => {
  const cart = await prisma.cart.findUnique({ where: { userId } });
  if (!cart) throw new Error("Cart not found");

  const product = await prisma.product.findUnique({ where: { id: productId } });
  if (!product) throw new Error("Product not found");

  if (product.stock < quantity) {
    throw new Error(
      `Insufficient stock. Only ${product.stock} items available.`,
    );
  }

  return await prisma.cartItem.update({
    where: { cartId_productId: { cartId: cart.id, productId } },
    data: { quantity },
  });
};
```

### 2. Error Handler Update: `./src/app.ts` (or wherever your error middleware is)

By default, an `Error` thrown in a service results in a 500 status. We want to catch these specific messages and return a 400. Check your error middleware; it should look something like this:

```ts
app.use((err: any, req: Request, res: Response, next: NextFunction) => {
  // If it's an "Insufficient stock" or "not found" error, treat as 400/404
  if (
    err.message.includes("Insufficient stock") ||
    err.message.includes("not found")
  ) {
    return res.status(400).json({ error: { message: err.message } });
  }

  console.error(err.stack);
  res.status(500).json({ error: { message: "Internal Server Error" } });
});
```
