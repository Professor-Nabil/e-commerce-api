### Error 2: `status` is an Enum, not a `string`

In your Prisma schema, `OrderStatus` is an **Enum** (e.g., `PENDING`, `COMPLETED`, `SHIPPED`). TypeScript won't let you pass a plain `string` because it wants to ensure you only use valid status values.

**Solution:** Import the generated `OrderStatus` type from `@prisma/client`.

```ts
import { prisma } from "../config/prisma.js";
import { AppError } from "../utils/appError.js";
import { OrderStatus } from "@prisma/client"; // <--- ADD THIS IMPORT

// ...

export const updateOrderStatus = async (
  orderId: string,
  status: OrderStatus,
) => {
  return await prisma.order.update({
    where: { id: orderId },
    data: { status },
  });
};
```
