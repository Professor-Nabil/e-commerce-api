Progress! Moving from `401` to `403` is a massive win. It means your **Authentication** is working (the token is valid), but your **Authorization** is blocking you.

The `403 Forbidden` tells us that the user is logged in, but their `role` is not `ADMIN`.

### The Problem

By default, when you register a user in your current `AuthService.registerUser`, Prisma likely defaults the `role` to `USER` (or whatever you set in `schema.prisma`). In your test, you registered a user, but they don't have the "Admin" badge required by `authorize(["ADMIN"])`.

### The Fix

We have two ways to solve this for the test:

#### Option A: Allow setting a role during registration (Easiest for testing)

Update your `src/services/auth.service.ts` to actually use a role if it's passed in the data:

```typescript
export const registerUser = async (userData: any) => {
  const hashedPassword = await bcrypt.hash(userData.password, 10);

  return await prisma.user.create({
    data: {
      email: userData.email,
      password: hashedPassword,
      role: userData.role || "USER", // Use the role if provided, else default
    },
    // ... select ...
  });
};
```

#### Option B: Update the Test to send the role

Now, in your `tests/product.test.ts`, tell the API that this specific user should be an admin:

```typescript
it("should allow an ADMIN to create a product", async () => {
  const adminCredentials = {
    email: "admin@test.com",
    password: "password123",
    role: "ADMIN", // Add this!
  };

  // 1. Register with ADMIN role
  await request(app).post("/api/auth/register").send(adminCredentials);

  // 2. Login
  const loginRes = await request(app)
    .post("/api/auth/login")
    .send(adminCredentials);
  const token = loginRes.body.token;

  // 3. Try to create the product
  const res = await request(app)
    .post("/api/products")
    .set("Authorization", `Bearer ${token}`)
    .send({
      name: "Mechanical Keyboard",
      price: 150,
      stock: 10,
    });

  expect(res.statusCode).toEqual(201);
});
```

---

### 🔍 Why did we get 403?

1. **Authentication (401):** "I don't know who you are." (Token missing or invalid).
2. **Authorization (403):** "I know who you are, but you aren't allowed to touch this." (User role is `USER`, but route requires `ADMIN`).

### One More Pro-Tip for your Schema

Check your `prisma/schema.prisma`. Ensure your `User` model has the role field defined correctly:

```prisma
model User {
  id    Int    @id @default(autoincrement())
  email String @unique
  password String
  role  String @default("USER") // Make sure this exists!
}
```
